# 绪论

本章的**基本概念**和**数据结构**定义属于概述内容，不在考纲中，属于背景补充！

## 基本概念

- **数据**是信息的载体，是对客观事物的符号表示，在计算机科学中是指所有能输入到计算中并被极速埃及程序处理的符号的总称。

- **数据** 是计算机程序加工的原料！

- **数据元素**是数据的基本单位，通常会被程序作为一个整体进行处理。一个数据元素可由多个数据项（数据项是构成数据元素的不可分割的最小单位）组成！

- **数据对象**是性质相同的数据元素的集合，是数据的一个子集。

---

## 数据结构

是相互之间存在一种或多种特定关系的数据元素的集合，包括逻辑结构和存储结构。

数据结构是一个二元组，**Data_Structure** = **(D, R)** 其中，**D** 代表数据元素的有限集合，**R** 是 **D** 上关系的有限集合。

### 逻辑结构

指数据之间的相互关系，分为四类：

1. 集合： 结构中的数据元素除了同属于一种类型外，再无其他关系。
2. 线性结构： 结构中的数据元素之间存在一对一的关系。
3. 树型结构： 结构中的数据元素之间存在一对多的关系。
4. 图状结构： 结构中的数据元素之间存在多对多的关系。

### 存储结构

指数据结构在计算机中的表示，又被称为数据的物理结构。

::: tip
一个算法的设计取决于逻辑结构，而算法的实现依赖于存储结构！
:::

1. 顺序存储方式，数据元素顺序存放，每个存储节点只有一个元素。存储位置反映数据元素间的逻辑关系。存储密度大，但某些操作效率较差。

2. 链式存储方式，每个存储节点除了包含数据元素信息外还包含一组指针。指针反映数据元素间的逻辑关系。该方式不要求存储空间连续，便于进行动态操作（插入、删除等），但存储开销增大（需要额外的空间来存储指针信息），并且无法进行随机存取。

3. 索引存储方式，除数据元素存储在一组地址连续的内存空间外，还需建立一个索引表，包含数据的关键字和对应的地址。

4. 散列存储方式，通过散列函数和解决冲突的方法，将关键字散列在连续的有限的地址空间内，并将散列函数的值解释为关键字所在元素的存储地址。特点是存储速度快，但只能按照关键字存储，无法自由随机存取。

## 数据类型

该概念和数据结构密切相关，最早出现在高级编程语言中，用于刻画程序操作对象的特性，在用高级程序语言编写的程序中，每个变量、常量或表达式都有一个它所属的确定的数据类型。

类型明显或隐含地规定了在程序执行期间变量或表达式所有可能取值的范围，以及在这些值上允许进行的操作。因此数据类型是一个值的集合和定义在这个值集合上的一组操作的总称。例如，C 中的整型变量，值存在范围（值的集合），并且支持数学运算（一组定义在整型这个集合上的操作）。

## 抽象数据类型

Abstract Data Type，简称 ADT，是指一个数学模型以及定义在该模型上的一组操作。抽象数据类型的定义仅取决于它的一组逻辑特性，而与其在计算机内部如何表示和实现无关，即不论其内部实现如何，只要其数学特性不变，就不影响其外部的使用。

::: tip

其实这里的抽象数据类型就是面向对象设计中的**类**思想，将普通的事物的特点抽象出来，组成一个抽象的概念用以描述具体的事物。

更确切点来说是**鸭子类型**，因为并不要内部实现如何。例如，你看到一个东西，它长得像鸭子，它还能下水，它还能嘎嘎叫，那么它就是鸭子！

:::

抽象数据类型可以使用三元组来进行表示：(D, S, P)

D 是数据对象，S 是 D 上的关系集，P 是对 D 的基本操作集。

**严蔚敏**的教材采用以下格式定义抽象数据类型：

```c
ADT 抽象数据类型名{
    数据对象：（数据对象的定义）
    数据关系：（数据关系的定义）
    基本操作：（基本操作的定义）
}ADT 抽象数据类型名
```

其中数据对象和数据关系的定义使用伪码表示，基本操作的定义格式为：

```c
基本操作名（参数表）
    初始条件：（初始条件描述）
    操作结果：（操作结果描述）
```

**基本操作**有两种参数，赋值参数仅仅用于提供输入值，引用参数以&开头，提供输入值并可返回操作结果。

**初始条件**表示操作执行前数据结果和参数应该满足的条件，不满足则操作失败并返回相应的出错信息。

**操作结果**说明来操作正常完成之后，数据结果的变化状况和应返回的结果。若初始条件为空，那么将其省略。

## 算法

算法是对特定问题求解步骤的一种描述，是指令的有限序列，其中每一条指令表示一个或多个操作。

### 特性

算法还具有下列5个重要特性：

1. 有穷性：算法必须在执行有限步之后结束，并且每一步均可在有限时间内完成。
2. 确定性：算法中的每一条指令必须要有确切的含义，不可出现歧义，即算法的运行结果可以复现（对于相同的输入，算法给出的输出是固定不变的）。
3. 可行性：算法必须可行，即算法中描述的操作都是可以通过已经实现的基本运算执行有限次来实现。
4. 输入：算法可以包含任意个输入，这取决于某个特定的对象的集合。
5. 输出：算法可以有任意个输出，每个输出均有输入相对应。

### 算法效率的度量

算法执行时间需通过依据该算法编制的程序在计算机上运行时所消耗的时间来度量。而度量一个程序的执行时间通常有两种方式。

#### 事后统计

因为很多计算机内部具有计时功能，不同的算法程序可以通过一个或者若干组相同的统计数据来分辨优劣。

但该方式存在两个缺陷，一个是必须先运行依据算法编制的程序，二是所得时间的统计量依赖于计算机的硬件、软件等环境因素，有时容易掩盖算法本身的优劣。

#### 事前分析估算

一个用高级程序语言编写的程序在计算机上运行时所消耗的时间取决于下列因素：

- 依据的算法选用的策略
- 问题的规模
- 书写程序的语言，语言级别越高，效率越低。
- 编译程序所产生的机器代码的质量
- 机器指令的执行速度

::: tip

很明显，一个算法使用不同的语言实现，或者用不同的编译程序进行编译，或者在不同的计算机上运行时，效率均会出现差异，这表明使用绝对的时间单位衡量算法的效率是不合适的。

:::

抛开那些不需要算法设计者需要注意的因素（例如计算机硬件、软件相关的因素），可以认为一个特定算法的运行工作量的大小，只依赖于问题的规模大小，一般使用 n 来表示。

一般情况下，算法中的**基本操作**重复执行的次数是问题规模 n 的某个函数 $f(n)$，算法的时间度量为：

$$ T(n) = O(f(n)) $$

表示随着问题规模 n 的增大，算法执行时间的增长率和 $f(n)$ 的增长率相同，被称为算法的渐进时间复杂度（asymptotic time complexity），简称**时间复杂度**。

::: tip

在算法分析中，基本操作（或原子操作）是指在算法执行过程中，所需时间可以认为是常数的操作。这些操作通常包括：

- 算术运算（如加、减、乘、除、取余、比较等）
- 数据移动（如赋值）
- 控制操作（如跳转）

:::

算法的**时间复杂度**还可能和输入数据集有关，以下是常见的算法时间复杂度：

$$O(1)<O(\log_{2}{n})<O(n)<O(n\log_{2}{n})<O(n^{2})$$

$$O(n^{2})<O(n^{3})<O(2^{n})<O(n!)<O(n^{n})$$

<!-- 增加习题 -->

**算法空间复杂度**$S(n)$定义为该算法所耗费的存储空间，它是问题规模n的函数，记为：

$$S(n)=O(g(n))$$

::: tip

一个程序在执行时除需要存储空间来存放本身所用的指令、常数、变量和输入数据外，还需要一些对数据进行操作的工作单元和存储一些为实现计算所需信息的辅助空间。

若输入数据所占空间仅仅取决于问题本身，和算法无关，则只需分析输入和程序之外的额外空间。

:::

**算法原地工作**是指算法所需的辅助空间为常量，即$O(1)$。
